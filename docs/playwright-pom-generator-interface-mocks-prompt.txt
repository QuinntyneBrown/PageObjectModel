You are working on the generator named PlaywrightPomGenerator.

Goal
Update PlaywrightPomGenerator so the generated Playwright control harness no longer uses "E2E bridge" naming. Use "interface mock" naming instead.

Context
The generated output currently resembles this structure:

- e2e/bridge/bridge-registry.ts
- e2e/bridge/bridge-providers.ts
- e2e/bridge/playwright-bridge.ts
- e2e/bridge/mocks/<interface-name>.mock.ts

The generated code currently uses names like:

- E2EBridgeRegistry
- e2eBridge
- E2EBridgeApi
- installE2EBridge
- provideE2EBridge
- window.__e2eBridge
- InterfaceBridge
- PlaywrightBridge
- BridgeCallRecord
- bridge-registry.ts
- bridge-providers.ts
- playwright-bridge.ts
- e2e/bridge

Rename Strategy
Remove "E2E" from generated API names and replace "service" vocabulary with "interface" vocabulary. The code is not just a transport bridge; it is a controllable mock harness for generated interface implementations.

Use this naming map:

- E2EBridgeRegistry -> InterfaceMockRegistry
- e2eBridge -> interfaceMocks
- E2EBridgeApi -> InterfaceMockApi
- installE2EBridge -> exposeInterfaceMocks
- provideE2EBridge -> provideInterfaceMocks
- window.__e2eBridge -> window.__interfaceMocks
- InterfaceBridge -> InterfaceMockHandle
- PlaywrightBridge -> InterfaceMocks
- BridgeCallRecord -> InterfaceCallRecord
- bridge-registry.ts -> interface-mock-registry.ts
- bridge-providers.ts -> interface-mock-providers.ts
- playwright-bridge.ts -> playwright-interface-mocks.ts
- e2e/bridge -> e2e/interface-mocks

Keep the top-level e2e output folder if that is how PlaywrightPomGenerator organizes test output. The request is to remove "E2E" from the generated bridge/mock API naming, not necessarily to move all generated files out of the e2e test area.

Generated Code Expectations
The registry file should generate roughly these public names:

- InterfaceCallRecord
- InterfaceMockApi
- InterfaceMockRegistry
- interfaceMocks
- exposeInterfaceMocks()
- Window.__interfaceMocks

The provider file should generate:

- provideInterfaceMocks()

The Playwright wrapper file should generate:

- InterfaceMockHandle
- InterfaceMocks

Generated mocks should import and use the renamed singleton:

```ts
import { interfaceMocks } from '../interface-mock-registry';

interfaceMocks.invoke('IBillsService', 'list', args);
return interfaceMocks.stream('IBillsService', 'list') as ReturnType<IBillsService['list']>;
```

Even if the underlying contracts are currently named IBillsService, IDashboardService, etc., generated harness comments and helper names should describe "interfaces" rather than "services" where practical. Keep actual contract names unchanged.
